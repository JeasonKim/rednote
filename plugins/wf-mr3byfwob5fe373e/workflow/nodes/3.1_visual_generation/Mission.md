## 职责

使用 `xiaohongshu-visual-notes` 技能，将文案内容转成适合小红书轮播的连续图文配图。

## 输入与参考

- `copywriting` 节点的 `outputs/note-copy.md`。
- `copywriting` 节点的 `outputs/image-content-brief.md`。
- workflow 层 `Context/account-positioning.md` 中的视觉偏好和账号语气。

## 执行步骤

// 先匹配模板候选

按照 `xiaohongshu-visual-notes` 技能的两阶段流程，先根据主题、目标、语气、信息密度和视觉需求匹配模板候选。候选阶段只展示模板编号、模板名称、一句话简介和适用场景。

// 再生成连续图片

模板确定后，按同一视觉风格生成 4-9 张连续图片。图片数量根据内容密度决定，不为凑页数制造重复页面。第 1 张为封面页，中间为核心内容页，最后 1 张为总结、行动建议或观点落点。

// 保持小红书阅读安全

默认使用 3:4 竖版画幅，保留顶部和底部安全边距。图中文字必须短、清晰、可读，标题优先控制在 18 个汉字以内，子标签优先控制在每个 6 个汉字以内。

## 产出

- `outputs/images/`：连续图片文件，文件名以两位数字编号开头，例如 `01-封面.png`、`02-核心观点.png`。
- `outputs/visual-summary.md`：记录已选模板、图片数量、每张图的主题和文件路径。

## 边界

本节点不改写完整发布文案，不制作 HTML 预览页，不向用户暴露完整原始 prompt 或内部出图 prompt。
