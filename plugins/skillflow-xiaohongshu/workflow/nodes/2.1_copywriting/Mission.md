## 职责

基于选题策划和账号定位，创作一篇适合小红书发布的单篇笔记文案，并为后续图文生成提供可拆页的内容素材。

## 输入与参考

- `topic_planning` 节点的 `outputs/topic-plan.md`。
- workflow 层 `Context/account-positioning.md`。
- 本次 task 中提供的产品、观点、素材、案例、链接或补充要求。

## 文案方法论

// 先确定平台化表达策略

根据选题类型选择适合的小红书写法。种草类内容优先围绕赛道、产品描述、目标人群、使用场景、真实体验、对比优势、购买建议和互动引导展开；知识类或经验类内容优先围绕痛点、方法、步骤、案例和总结展开。

// 标题先做多候选再收敛

生成 5 个标题候选，标题控制在 20 个汉字以内，尽量口语化、有情绪、有悬念或明确获得感。可使用正向刺激、负向刺激、痛点刺激、低门槛获得感、对比反差、金句或热点借势等方式，但不得制造虚假承诺。

// 正文按小红书阅读习惯组织

正文建议控制在 400-500 字；如果 task 明确要求长文或短文，可按要求调整。每段尽量短句、口语化，允许适量 emoji 增强节奏，但不能影响阅读。正文应包含开篇痛点或场景、核心内容展开、真实感或证据支撑、总结推荐或行动建议、评论区互动引导。

// 标签兼顾搜索与转化

标签应覆盖核心关键词、关联长尾词、高转化词和行业热搜词。标签必须与正文和选题一致，不能堆砌无关热词。

## 产出

- `outputs/note-copy.md`：完整可发布文案，包含标题候选、推荐标题、正文、标签、评论区引导。
- `outputs/image-content-brief.md`：供配图节点使用的图片页内容简报，包含封面钩子、建议页数、每页主题和每页文字重点。

## 边界

本节点不调用图像生成，不制作网页预览，不直接输出内部图片生成 prompt。
